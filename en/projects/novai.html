NovAI - Interactive Novel Reader

Overview

NovAI is a full-stack web application that transforms online novels into an accessible listening experience. It combines web scraping technology with text-to-speech capabilities to allow users to both read and listen to novels with real-time paragraph highlighting.

The project features a React frontend with Material UI for the interface and a Python backend (FastAPI/Flask) that handles novel retrieval and processing. This separation of concerns allows for a responsive user experience while efficiently managing data processing on the server.

Story Behind the Project

As an avid webnovel reader, I found myself manually copying and pasting chapters from websites into TTS tools just to listen to them. This process started with tab-switching and copy-pasting each chapter individually, then evolved into basic automation using AHK scripts. Eventually, I discovered web scraping, which let me batch-download chapters into a text file—speeding up the process dramatically.

That workflow worked, but it lacked elegance. I wanted something smoother, faster, and more accessible—so I built NovAI. With a full frontend and backend pipeline, chapter scraping became instant, and listening became seamless. What began as a personal solution to a repetitive task turned into a tool designed for comfort and efficiency.

Key Features

  • Novel Content Retrieval: Search and fetch novels by keyword from online sources
  • Interactive Text-to-Speech: Listen to novels with adjustable reading speed (0.5x to 2.5x)
  • Real-time Visual Tracking: Paragraphs are highlighted as they’re being read
  • Full Playback Controls: Play, pause, and stop functionality
  • Content Management: Copy text to clipboard or download as TXT files
  • Responsive Design: Optimized for various screen sizes
  • Content Caching: Server-side caching for improved performance

Tech Stack

Frontend

  • React: Component-based UI development
  • Material UI: Design system for consistent styling and theming
  • TTS-Reader.com Engine: External text-to-speech service integration
  • React Hooks: State management and component lifecycle
  • Fetch API: Communication with backend services

Backend

  • FastAPI/Flask: API endpoints for content retrieval
  • aiohttp/BeautifulSoup4: Asynchronous web scraping
  • SQLite: Content caching database
  • asyncio: Asynchronous I/O framework
  • Pydantic: Data validation and settings management

Architecture

The application follows a client-server architecture with external services:

  1. Client (React): Handles user interface and visualization
  2. TTS-Reader.com Engine: Provides high-quality text-to-speech functionality
  3. Server (FastAPI/Flask): Manages novel discovery, content scraping, and text processing
  4. Database (SQLite): Caches scraped content to improve performance and reduce load on source websites

Backend Implementation

The Python backend handles the heavy lifting of content retrieval:

  • Novel Search: Finds the correct novel based on user keywords
  • Content Scraping: Extracts chapter content from web sources
  • Text Processing: Cleans and formats text for optimal reading/listening
  • Response Formatting: Returns structured data with both raw and formatted content
  • Caching System: Stores previously retrieved content to improve performance
async def get_text(keyword, chapter, number):
    async with aiohttp.ClientSession(headers=headers) as session:
        urls, novel_title, novel_image = await get_urls(session, number, chapter, keyword)

        tasks = [get_page_content(session, url) for url in urls]
        texts = await asyncio.gather(*tasks)

        text = '\n'.join(texts)
        text = preprocess(text)

        text_array = [par.strip() for par in text.split("\n")]
        text_formatted = "".join([f'<p id="par{i}">{par}</p>' for i, par in enumerate(text_array)])

        return {
            'text': text,
            "title": novel_title,
            "image": novel_image,
            "formatted": text_formatted,
            "array": text_array
        }
